# cid-frontend guidance

Este repo es Angular.

Al trabajar aquí:
- Prioriza claridad de componentes, tipado fuerte y consistencia con el patrón actual.
- Revisa routing, guards, interceptors, formularios y manejo de estado antes de proponer cambios.
- Si una tarea toca UX o performance, consulta `angular_expert` y `angular_optimizer`.
- Para tareas complejas, primero planifica con `architect` y `angular_expert`, luego ejecuta y revisa con agentes mini.
- Toda implementación debe ser revisada por `code_reviewer` y `security_auditor`.
- No agregues dependencias nuevas salvo que sean claramente necesarias.
